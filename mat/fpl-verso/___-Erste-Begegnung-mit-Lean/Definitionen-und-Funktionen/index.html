1.5. Definitionen und Funktionen🔗

def definiert ein Symbol. Es kann für einen Wert, eine Funktion oder auch einen Typ stehen.

def hello := "Hallo" hello : String#check hello "Hallo"#eval hello def adams := 42 adams : Nat#check adams 42#eval adams def addOne x := x + 1 addOne (x : Nat) : Nat#check addOne 42#eval addOne (adams - 1)

Funktionen können mehrere Argumente haben. Bei ihrer Auswertung wird per Currying bei einer Funktion der Arität n mit dem ersten Argument eine Funktion der Arität (n-1) ermittelt usw. bis die letzte einstellige Funktion zum Ergebnis ausgewertet werden kann.

Hinweis: Der Begriff Currying kommt daher, dass Haskell Brooks Curry die Umwandlung einer Funktion mit mehreren Variablen in eine Folge von unären Funktionen 1958 beschrieben hat. Tatsächlich könnte man auch von Schönfinkeln sprechen, denn Moses Schönfinkel hat dies schon 1924 beschrieben.

Haben wir eine Funktion f mit drei Variablen des Typs Nat, dann hat sie folgenden Typ:

Nat → Nat → Nat → Nat

und → ist rechts-assoziativ, d.h. der Typ ist explizit:

Nat → (Nat → (Nat → Nat))

Also ist f n₁ eine Funktion mit 2 Variablen, d.h. vom Typ Nat → (Nat → Nat) und f n₁ n₂ hat den Typ Nat → Nat, ist also eine unäre Funktion. Es ist also:

f n₁ n₂ n₃ = ((f n₁) n₂) n₃

Beispiel einer zweistelligen Funktion:

def swapSign (n : Nat) (b : Bool) : Int := if b then n else -n swapSign (n : Nat) (b : Bool) : Int#check swapSign swapSign : Nat → Bool → Int#check (swapSign) swapSign : Nat → Bool → Int#check @swapSign swapSign 42 : Bool → Int#check swapSign 42 swapSign 42 true : Int#check swapSign 42 true -42#eval swapSign 42 false

Die Auswertung von Funktionen in Lean ist strikt. Das bedeutet, dass zunächst die Ausdrücke der übergebenen Argumente ausgewertet werden und dann erfolgt die Anwendung der Funktion.

-42#eval swapSign (39 + 3) (2 == 3)

1.5.1. Funktionen höherer Ordnung🔗

Funktionen höherer Ordnung sind solche, die Funktionen als Argumente haben oder zu Funktionen auswerten.

Als Beispiel definieren wir die Funktion comp für die Komposition zweier Funktionen Nat → Nat: g ∘ f, „g nach f":

def comp (g : Nat → Nat) (f : Nat → Nat) : (Nat → Nat) := fun x => g (f x) def f x := x + 1 def g x := x * 2 def g_kringel_f := comp g f g_kringel_f : Nat → Nat#check g_kringel_f 4#eval g_kringel_f 1

Hierbei haben wir gleich gesehen, wie man anonyme Funktionen in Lean definiert: durch einen Lambda-Ausdruck beginnend mit fun, wobei der Körper der Funktion hinter => kommt.

42#eval (fun x => x * 2) 21 42#eval (fun x y => x + y) 41 1

Es gibt auch eine andere Ausdrucksweise für einfache Lambdas (dabei wird · durch die Tastenkombination \ mit . erzeugt):

fun x => x + 1 : Nat → Nat#check (· + 1) 1#eval (2 - ·) 1 def h (x y z : Nat) := x + y + z fun x1 x2 => h x1 1 x2 : Nat → Nat → Nat#check (h · 1 ·) 7#eval (h · 1 ·) 3 3

[01.11] Schreiben Sie eine Funktion swap, die als Argument eine Funktion α → β → γ hat und eine Funktion zurückgibt, die die Reihenfolge der beiden Argumente der übergebenen Funktion vertauscht.

swap sollte also zum Beispiel so wirken:

#eval (swap (· - · ) 3 (2 : Int ))

Anmerkung: Die Funktion, die die Argumente vertauscht, heißt in Lean flip.

[01.12] Definieren Sie eine Funktion nfach die als Argumente eine einstellige Funktion natürlicher Zahlen, eine Zahl n sowie einen Ausgangswert k hat:

def nfach (f: Nat → Nat): Nat → Nat → Nat

nfach soll die Funktion f n-mal auf den Wert k anwenden.

Ist mal2 definiert als:

def mal2 := (· * 2)

dann soll gelten:

#eval mal2 (mal2 (mal2 5)) == nfach mal2 3 5

Anmerkungen:

  • Diese Funktion heißt in Lean Nat.iterate.

  • Für die Definition dieser Funktion brauchen Sie Pattern Matching, das im nächsten Kapitel genauer erläutert wird.

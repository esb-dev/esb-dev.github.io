8.8. Beispiel State🔗

Die Monade StateM erlaubt es nicht nur Daten im Kontext zu lesen, sondern sie auch zu ändern. Wir nehmen als Beispiel einen Stack.

Wenn man einen Stack mit den Operationen push und pop darauf hat, würde man das in einer objektorientierten Sprache wie Java etwa so machen:

Deque<Integer> stack= new ArrayDeque<>();
boolean result = stack.empty();
System.out.println("Ist der Stack leer? " + result);
// Werte auf den Stack legen
stack.push(1);
stack.push(2);
stack.push(3);
stack.push(42);
stack.push(43);
stack.pop;
// Stack ausgeben
System.out.println("Werte im Stack: " + stack);

Das ergibt folgende Ausgabe:

Ist der Stack leer? true
Werte im Stack: [1, 2, 3, 42]

Da das Objekt in Java zustandsbehaftet ist, ändern die Funktionen push und pop den Zustand des Stacks.

Wenn wir nun dasselbe Beispiel in Lean machen wollen, sind wir mit der Tatsache konfrontiert, dass Lean keine zustandsbehafteten Objekte kennt.

Wenn wir den Stack durch eine Liste repräsentieren, dann könnten wir folgende Implementierung für den Stack machen:

structure Stack (α : Type) where st : List α deriving Repr def Stack.new (α : Type) : Stack α := { st := (List.nil : List α) } def Stack.push (stack : Stack α) (a : α) : Stack α := let l := stack.st Stack.mk (a :: l) def Stack.pop? (stack : Stack α) : (Option α × Stack α) := match stack.st with | [] => (none, stack) | a :: as => (some a, Stack.mk as) def Stack.empty (stack : Stack α) : Bool := stack.st.length == 0

Mit dieser Implementierung könnte man das obige Beispiel folgendermaßen in Lean nachvollziehen:

namespace Stack def stackExample : Stack Nat := let stack := new Nat let stack1 := stack.push 1 let stack2 := stack1.push 2 let stack3 := stack2.push 3 let stack4 := stack3.push 42 let stack5 := stack4.push 43 Prod.snd (pop? stack5) { st := [42, 3, 2, 1] }#eval stackExample 
{ st := [42, 3, 2, 1] }

Wie wir sehen, brauchen wir für jeden Schritt immer den vorherigen Zustand und geben auch einen neuen Zustand zurück. Wir könnten also das Beispiel auch so schreiben:

def stackExample' : Stack Nat := let stack0 := new Nat let stack1 := push stack0 1 let stack2 := push stack1 2 let stack3 := push stack2 3 let stack4 := push stack3 42 let stack5 := push stack4 43 let stack6 := Prod.snd (pop? stack5) stack6 { st := [42, 3, 2, 1] }#eval stackExample' 
{ st := [42, 3, 2, 1] }

Offenbar müssen wir jeder Funktion immer den bisherigen Stack mitgeben und dann den zurückgegebenen neuen Stack an die nächste Funktion weiterreichen.

Wie man sieht funktioniert das mit push gut, aber nicht mit pop?, weil wir hier ein Paar als Ergebnis haben.

Wir müssen also bei beiden Funktionen ein Paar zurückgeben:

namespace Stack' def push (stack : Stack Nat) (a : Nat) : Unit × Stack Nat := ((), Stack.push stack a) def pop? (stack : Stack Nat) : Option Nat × Stack Nat := let res := Stack.pop? stack match res.fst with | none => (none, res.snd) | some a => (some a, res.snd)

Damit können wir das Beispiel so formulieren:

def stackExample' : Stack Nat := let stack0 := Stack.new Nat let stack1 := (push stack0 1).snd let stack2 := (push stack1 2).snd let stack3 := (push stack2 3).snd let stack4 := (push stack3 42).snd let stack5 := (push stack4 43).snd let stack6 := (pop? stack5).snd stack6 { st := [42, 3, 2, 1] }#eval stackExample' 
{ st := [42, 3, 2, 1] }
end Stack'

Wie man sieht, muss man immerzu den Stack als Argument an die nächste Funktion weitergeben. Die Grundidee der Monade StateM besteht darin, das Weitergeben des Zustands von der eigentlichen Funktion zu trennen: Die Monade StateM gibt (bildlich gesprochen) den Stack jeweils an die nächste Funktion weiter.

Eine zustandsändernde Aktion ist eine Funktion, die den aktuellen Zustand bekommt und einen Ergebniswert zusammen mit dem neuen Zustand zurückgibt. Die Monade StateM kann das Weitergeben des Zustands für uns übernehmen. Sie gibt genau genommen nicht den Zustand als Wert weiter, sondern wendet eine zustandsändernde Aktion an.Diese Erläuterung ist sehr knapp und hat nur die Absicht, verständlich zu machen, wie man Zustandsmonaden anwendet. Eigentlich ist es zum tiefen Verständnis erforderlich, dass man analysiert, wie bind und pure für Zustandsmonaden definiert sind.

Wie ist pure definiert?

def pure (x : α) : StateM σ α :=
  fun s => pure (x, s)

pure gibt den Wert x zurück ohne den Zustand s zu verändern.

Wie ist bind definiert?

def bind (st : StateM σ α)
    (f : α → StateM σ β) : StateM σ β :=
  fun s => do let (x, s') ← st s; f x s'

st ist die Zustandsmonade und f eine Funktion, die eine Zustandsmonade zurückgibt.

Gegeben ein aktueller Zustand s, (der später übergeben wird, wenn wir die Monade repräsentiert durch x >>= f verwenden --- siehe Parameter s der Funktion in der Definition von bind),

  1. dann ergibt st s einen Wert x sowie einen neuen Zustands s'.

  2. Dann wenden wir f an: f x, was eine neue Zustandsmonade ergibt.

  3. f x s' ergibt dann einen neuen Wert b sowie einen neuen Zustand s''.

st >>= f repräsentiert also die Berechnung von s zu (b, s''). Wenn man das mehrfach macht, dann komponiert man die Berechnungen und der jeweilige Zustand wird "hinter den Kulissen" weitergereicht. Am Ende erhalten wir eine Gesamtberechnung, die wir dann mit einem initialen Zustand "laufen lassen" können.

Wir wenden das auf unser Beispiel mit dem Stack an und ersparen uns das mühsame Weitergeben des jeweiligen Zustands des Stacks:

namespace StateStack def push (a : Nat) : StateM (Stack Nat) Unit := do modify fun stack => Stack.push stack a

modify nimmt den bisherigen Zustand und wendet eine Funktion darauf an, die den neuen Zustand zurückgibt. modify entspricht do set (f (← get)), d.h. liest den bisherigen Zustand mit get, wendet die übergebene Funktion an und setzt den neuen Zustand.

def pop : StateM (Stack Nat) (Option Nat) := do let stack ← get let res := Stack.pop? stack let stack' := res.snd set stack' return res.fst

get und set verwenden den Zustand, den die Monade weiterreicht, lesend bzw. schreibend.

def stackExample : StateM (Stack Nat) Unit := do push 1 push 2 push 3 push 42 push 43 let _ ← pop return () ((), { st := [42, 3, 2, 1] })#eval stackExample.run (Stack.new Nat) 
((), { st := [42, 3, 2, 1] })
((), { st := [42, 3, 2, 1] })#eval stackExample (Stack.new Nat) 
((), { st := [42, 3, 2, 1] })
{ st := [42, 3, 2, 1] }#eval (stackExample (Stack.new Nat)).snd 
{ st := [42, 3, 2, 1] }
[42, 3, 2, 1]#eval (stackExample (Stack.new Nat)).snd.st 
[42, 3, 2, 1]

Wie bei ReaderM gibt es auch hier die Funktion run, mit der der Startzustand übergeben wird und dann die Funktion ausgeführt wird. In unserem Beispiel starten wir mit einem leeren Stack natürlicher Zahlen.

Details zu Zustandsmonaden finden Sie in The Lean Language Reference: 14.5.4 State.

Noch ein Beispiel: Am Beispiel eines Zählers wollen wir besser verstehen, dass die Zustandsmonade nicht wirklich den Zustand "weitergibt", sondern eine zustandsändernde Funktion anwendet, einen "state transformer".

namespace counter -- ST steht für einen "state transformer", der einen Zustand `s` -- bekommt und ein Paar `(a, s)` zurückgibt. abbrev ST (α : Type) := StateM Nat α -- Wir definieren eine Funktion, die keinen Wert liefert, -- sondern nur den Zustand verändert def inc : ST Unit := do let n ← get set (n+1) -- oder: modify (· + 1) -- Wir definieren eine Funktion, die den aktuellen Wert -- des Zustands liefert def getCnt : ST Nat := do let n ← get return n -- oder def getCnt : ST Nat := get -- Ein Beispiel: def inc3 : ST Nat := do inc inc inc getCnt (3, 3)#eval inc3.run 0 
(3, 3)
(10, 10)#eval inc3.run 7 
(10, 10)
end counter

[08.1] Gegeben seien die beiden folgenden Funktionen:

def plus (n : Nat) : Nat → Nat := (· + n) def minus (n : Nat) : Nat → Nat := (· - n)

Mit ihnen kann man etwas umständlich arithmetische Ausdrücke mit plus und minus auswerten, zum Beispiel

8#eval (plus 5) 3 
8
8#eval (plus 3) 5 
8
0#eval (minus 3) 3 
0
0#eval (minus 5) 3 
0
42#eval ((plus 21) ∘ (plus 21)) 0 
42
0#eval ((minus 42) ∘ (plus 41)) 1 
0
0#eval ((minus 43) ∘ (plus 41)) 1 
0
21#eval ((plus 21) ∘ (minus 21)) 0 
21

Wir möchten nun gerne in den mit [*] gekennzeichneten Beispielen unterscheiden können, ob das Ergebnis exakt 0 ist oder durch die Definition von Nat.sub zustande kam.

Definieren Sie zwei Funktionen mit folgender Signatur:

def declaration uses 'sorry'pluso (n : Nat) : Nat → Option Nat := sorry def declaration uses 'sorry'minuso (n : Nat) : Nat → Option Nat := sorry

und machen Sie dieselben Beispiele mit diesen Funktionen. Und schließlich definieren Sie Funktionen

def declaration uses 'sorry'plusl (n : Nat) : Nat → List Nat := sorry def declaration uses 'sorry'minusl (n : Nat) : Nat → List Nat := sorry

die auch den Unterschied der mit [*] gekennzeichneten Beispiele darstellen.

[08.2] Schreiben Sie eine Funktion listAdd (l₁ l₂ : List Nat) : List Nat, die eine Liste ergibt, deren Elemente die Addition aller Paare der Elemente in den beiden Listen l₁ und l₂ sind.

Beispiel:

#eval listAdd [1, 2] [1, 2, 3, 4, 5]

[08.3] Unter einem pythagoreischen Tripel versteht man 3 natürliche Zahlen bei denen die Summe der Quadrate der beiden kleineren Zahlen gerade das Quadrat der größten Zahl ergibt. Das erste Beispiel ist (3, 4, 5).

Schreiben Sie eine Funktion

def declaration uses 'sorry'pythaTripel (n : Nat) : List (Nat × Nat × Nat) := do sorry

die alle pythagoreischen Tripel mit Zahlen bis zur Schranke n auflistet.

[08.4] Schreiben Sie ein Programm main : IO Unit, das eine Zeile von stdin einliest und die Zahl der Worte der Eingabe ausgibt.

[08.5] Schreiben Sie eine Funktion addIO : IO (Option Int), die nach zwei Zahlen fragt und dann deren Summe als ein some zurückgibt. Es kann sein, dass die Eingabe keine gültigen Zahlen sind, dann wird ist das Ergebnis none.

Verwenden Sie die Funktion in folgendem main:

def main : IO Unit := do

let stdout ← IO.getStdout

let result ← addIO

stdout.putStrLn s!"{result}"

Wenn man das Programm ausführt, erhält man beispielhaft ein Ergebnis wie:

Geben Sie Zahl 1 ein:

4

Geben Sie Zahl 2 ein:

38

(some 42)

oder

Geben Sie Zahl 1 ein:

s

Geben Sie Zahl 2 ein:

3

none

[08.6] Eine Funktion zum Aufstellen der Wahrheitstafel für eine Boolesche Funktion findet man im Internet, siehe Lean 99: Ninety-Nine Lean Problems, Problem 48 (Man kann dort nicht nur die Aufgabe sehen, sondern mit einem Link ganz oben rechts auch die vorgeschlagene Lösung). In der Aufgabe wird List eine Instanz der Typklasse Monad, wir können aber auch einfach List aus der Mathlib nehmen:

namespace tt def Arity : (n : Nat) → Type | 0 => Bool | n + 1 => Bool → Arity n def ttable (n : Nat) (p : Arity n) : List (List Bool) := match n with | 0 => [[p]] | n + 1 => do let b ← [true, false] let result ← ttable n (p b) |>.map (b :: ·) return result

Beispiele:

[[true, false], [false, true]]#eval ttable 1 (fun a => !a) 
[[true, false], [false, true]]
[[true, true, true], [true, false, true], [false, true, true], [false, false, false]]#eval ttable 2 (fun a b => a || b) 
[[true, true, true], [true, false, true], [false, true, true], [false, false, false]]
[[true, true, true, true], [true, true, false, false], [true, false, true, false], [true, false, false, false], [false, true, true, false], [false, true, false, false], [false, false, true, false], [false, false, false, false]]#eval ttable 3 (fun a b c => a && b && c) 
[[true, true, true, true], [true, true, false, false], [true, false, true, false], [true, false, false, false],
  [false, true, true, false], [false, true, false, false], [false, false, true, false], [false, false, false, false]]
end tt
